# ColorShot 개발 계획표

> 색채 기반 일상 기록 서비스 — "찍으면 예뻐지는 자동 팔레트" + "색을 이야기로 돌려주는 AI 리캡".
> 현 스택: **Expo 57 / RN 0.86 / React 19 / NativeWind(Tailwind)** — 유지.
> ⚠️ 코드 작성 전 반드시 Expo v57 버전 문서 확인 (`AGENTS.md`).

---

## 0. 지금까지 (as-is)

| 영역 | 상태 | 파일 |
| --- | --- | --- |
| 카메라/갤러리 입력 | ✅ | `App.tsx` (expo-image-picker) |

## 2. Phase 1 — "줍고, 쌓인다" (MVP 완성)

목표: 친구 0명·서버 0으로 **색 줍기 → 저장 → 이번 주 색 → 3뷰** 완결.

### 2-1. 색 줍기 (dot 선택) — 최우선, 정체성
- [ ] `extractColors`가 색마다 **화면 좌표(centroid)** 반환하도록 확장
  - median-cut 버킷에 픽셀의 (x,y)도 함께 담기 → 버킷 평균좌표 계산 → 원본 표시 크기로 스케일
- [ ] 이미지 위에 각 색 **dot 오버레이** (절대배치 View, dot 색 = 추출색)
- [ ] dot **원탭 → 즉시 수집** (확인 버튼 없음), 여러 개 연속 탭 가능
- [ ] 안 주워도 넘어가기 허용
- **검증:** "자동으로 다 뽑기"가 아니라 "내가 골라 주웠다" 감각이 나는가

### 2-2. 저장 & 자동 메타
- [ ] expo-sqlite 스키마: `CollectedColor / Session / (MoodCheck)`
- [ ] 색 저장 시 `createdAt` 자동, `lat/lng` 자동(선택, expo-location)
- [ ] 저장 시 파생값 계산: `hue`, `saturation`, `warmth`(따뜻/차가움)
- **검증:** 앱 껐다 켜도 주운 색이 남아있는가

### 2-3. 3-뷰 (한 데이터, 세 보기)
- [ ] 색상 보기 — hue 군집 무드보드 (현 ColorPalette 확장)
- [ ] 시간순 보기 — "이번 주 주운 색" 스트립
- [ ] 무드 보기 — warm/cool·채도 축
- **검증:** 색 한 번 주우면 세 뷰에 전부 반영되는가

### 2-4. 카메라 우선 UX 정리
- [ ] 플로팅 바에서 **촬영을 기본(주 버튼)** 으로, 갤러리는 보조
- [ ] 기분 체크 1탭(선택·스킵 우선) 세션 전후 배치

## 3. Phase 2 — AI 주간 리캡

- [ ] 주간 집계(온디바이스): 색 목록 + 요일/시간 + warm/cool·채도 분포 + 메모
- [ ] 서버 라우트(Claude 프록시), **키는 서버 전용** (앱에 넣지 않음)

- **검증:** 서로 다른 두 주가 확실히 다른 문장으로, 단정 없는 톤으로 나오는가

## 5. 디자인 방향 (design language)

컨셉이 "도파민 SNS 반대편의 회복"이므로, 디자인 원칙도 그걸 따른다.

**원칙**
- **색이 유일한 주인공** — UI 자체는 무채색/뉴트럴, 강조색은 오직 유저가 주운 색.
- **조용함** — 빨간 알림 배지·숫자 카운트·무한 스크롤 없음. 불안 유발 요소 제거.
- **여백** — 빽빽함 대신 넉넉한 공백. "느리고 의도적인" 경험.
- **둥근 형태·부드러운 그림자** — 현재 rounded-3xl 톤 유지(좋음).
- 다크모드 대응(색을 어두운 캔버스에 얹으면 더 살아남).

**디자인 필요한 화면 (우선순위순)**
1. **줍기 화면** — 사진 위 dot 오버레이 + 수집 인터랙션 (가장 중요, 신규)
2. **홈/3뷰** — 색상·시간순·무드 전환 (현 화면 확장)
3. **주간 리캡 카드** — 팔레트 + AI 텍스트 (공유 욕구 자극, 마케팅 포인트)
4. **기분 체크 1탭** — 초경량, 스킵 우선
5. 온보딩(최소), 빈 상태(텅 비어도 예쁘게)
